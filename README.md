## рҹҢҢ What Is This?

**Satellite-Driven Socioeconomic Intelligence** is a **production-grade AI analytics platform** that transforms raw satellite imagery into actionable socioeconomic intelligence вҖ” in under 200 milliseconds.

It doesn't just show you a map. It **reads the Earth like a book** вҖ” decoding nightlights, road density, urban heat, vegetation patterns, and spectral signatures to answer the question that matters most to policymakers, investors, researchers, and NGOs:

> *"How developed is this place вҖ” and where is it headed?"*

Built on a **pipeline-calibrated Gradient Boosting model** trained on 14,460+ real-world city samples, this platform covers **44 Indian cities** (from Bangalore's tech corridors to the deep rurals of Bihar and Arunachal Pradesh) and **101 global regions** across 6 continents.

## рҹҡҖ Quick Start

### Prerequisites

```bash
Python 3.9+   # Required
pip           # Package manager
~500 MB disk  # For dependencies + model
```

<br/>

### вҡЎ One-Command Launch

```bash
# Clone / unzip the project
cd Satellite-Socioeconomic-Intelligence

# Make runnable and go
chmod +x run.sh && ./run.sh
```

That's it. The script will:
1. вң… Check Python version
2. вң… Install all dependencies automatically
3. вң… Pre-train the ML model (once, ~8 seconds)
4. вң… Start the FastAPI backend on port `8000`
5. вң… Start the Streamlit frontend on port `8501`
6. вң… Monitor and auto-restart crashed services

<br/>

### рҹҗҚ Python Launcher (Cross-Platform)

```bash
# Full stack
python3 start.py

# Backend only (API development)
python3 start.py --backend-only

# Frontend only (if backend is elsewhere)
python3 start.py --frontend-only

# Skip pip install
python3 start.py --no-install
```

<br/>

### рҹ”§ Manual (Two Terminals)

```bash
# в”Җв”Җ Terminal 1: FastAPI Backend в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
```

```http
file: <satellite_image.jpg>
```

```bash
     -F "file=@satellite_image.jpg"
```

**Python:**
```python
import requests

with open("satellite.jpg", "rb") as f:
    response = requests.post(
        "http://localhost:8000/predict",
        files={"file": ("satellite.jpg", f, "image/jpeg")},
        params={"region_name": "Mumbai Western Suburbs"}
    )

result = response.json()
print(f"Score: {result['percentage']:.1f}/100")
print(f"Category: {result['category']}")
print(f"Confidence: {result['confidence']*100:.0f}%")
```

### Training Data Generation

```python
# 360 cities from the global database
# Г— 40 spatial augmentations each (small lat/lon jitter + pipeline noise)
# + 100 extreme-end synthetic anchors
# = 14,460 training samples

for city in CITY_DATABASE:                    # 360 cities
    for aug in range(40):                     # 40 augmentations
        dlat, dlon = rng.normal(0, 0.05, 2)  # spatial jitter
        X = extract_features(city.lat + dlat, city.lon + dlon)
        y = city.score + rng.normal(0, 0.01) # label noise
```

### Why Pipeline-Calibration Matters

Most ML models are trained on generic synthetic data that doesn't match the actual feature extractor output distribution. This causes **systematic bias** вҖ” the model sees features during training that it never sees during inference.

Our solution: **train directly on pipeline output**. Every training sample is generated by calling `extract_features_from_region()` вҖ” the exact same function used during inference. Zero distribution shift. Zero mismatch.

## рҹ”¬ How Feature Extraction Works

### From Satellite Image (Upload Mode)

```python
def extract_features_from_image(image_bytes: bytes) -> np.ndarray:

    img = Image.open(io.BytesIO(image_bytes)).convert("RGB")
    r, g, b = arr[:,:,0], arr[:,:,1], arr[:,:,2]
    lum = 0.299*r + 0.587*g + 0.114*b   # perceptual luminance

    features = [
        mean(lum > 0.60),                 # night-light intensity
        clip(mean(NDVI(r,g)) * 0.5 + 0.5),  # vegetation index
        mean(grey_similar & mid_bright),  # built-up area
        mean(edge_magnitude > 0.12),      # road density
        mean(blue_dominant & dark),       # water bodies
        mean(red_dominant & warm),        # urban heat
        mean([std(r), std(g), std(b)]),   # spectral variance
        texture_entropy(lum),             # texture complexity
        mean(edge_magnitude > 0.07),      # edge density
        mean(lum > 0.68),                 # brightness
        mean(r > 0.58),                   # infrared proxy
        0.4*nightlight + 0.3*built + 0.3*road  # settlement index
    ]
```

This platform is built for **research, policy screening, and investment due diligence**. Important limitations:
